Phase 0这技术到底是啥？

一句话定义

DeepSeek-R1 是一个完全跳过传统教学，让 AI 在"试错"中自己摸索出推理能力的方法。它不用人工标注的解题步骤，而是通过强化学习（RL），让模型在一次次尝试中学会怎么思考。

🎯 生活类比：教孩子学数学，不直接教步骤

想象一下两种教孩子学数学的方式：

  • 传统方法（SFT）：老师把解题步骤写得清清楚楚，孩子照着抄——学会了，但不会举一反三
  • DeepSeek-R1 的方法：老师只告诉孩子"答对了有奖励，答错了没奖励"，让孩子自己尝试——孩子会摸索出各种解题方法，甚至发明老师没教过的技巧

核心价值：解决了什么问题？

这个方法听起来很"佛系"，但它解决了一个核心痛点：

  • 打破人类思维天花板：传统方法受限于人类怎么思考，AI 只能模仿人类；RL 让 AI 自己探索，可能发现比人类更聪明的思考方式
  • 减少人工标注成本：不用人工写解题步骤，只需要知道"答案对不对"——省了巨量人力
  • 自主进化：模型会自己发展出验证、反思、探索替代方案等高级推理行为

Phase 1以前的技术为什么不行？

传统方法 1：Chain-of-Thought Prompting

CoT 就是让模型"一步步想"，比如在 prompt 里写"Let's think step by step"。这个方法有效，但有问题：

CoT 的死穴

  • 依赖人工设计：prompt 模板是人工写的，带着人类的认知偏差
  • 无法探索新方法：模型被限制在人类设计的框架里，无法发现更优的推理路径
  • 上下文浪费：推理步骤占用了大量 token，留给答案的空间就少了

传统方法 2：监督微调（SFT）

SFT 就是给模型看大量"问题+标准答案"的例子，让它学会怎么解题。这个方法的问题更严重：

SFT 的死穴

  • 标注成本极高：需要人工写详细的推理步骤，数学题、编程题、逻辑题——每道都要人写
  • 受限于人类水平：模型最多学到人类会的方法，无法超越人类
  • 认知偏差：标注数据带着人类的主观判断，模型会继承这些偏见

传统方法 3：RL + SFT 两阶段训练

有些研究尝试先用 RL 训练推理能力，再用 SFT 对齐人类偏好。这个方法的问题是：

  • 流程复杂：要分两个阶段训练，中间还要准备数据，工程复杂度高
  • 可能互相抵消：RL 阶段学到的推理能力，可能在 SFT 阶段被"对齐"掉

🎯 类比：学开车，教练 vs 自己练

  • CoT：教练坐在副驾驶，每一步都告诉你怎么开——安全但学不会灵活应变
  • SFT：给你看 1000 个"完美驾驶视频"，你照着模仿——学会了标准动作，但遇到突发情况不会处理
  • RL：把你扔到空旷场地，让你自己开，撞了就扣分，开得好就加分——你会摸索出各种技巧，甚至发明新方法

Phase 2DeepSeek-R1 到底是怎么做到的？

核心思想：一句话

DeepSeek-R1 用强化学习（RL）让模型自主学会推理，完全跳过传统的 SFT 阶段。模型在一次次尝试中摸索出解题方法，就像人类在试错中学习。

第一步：DeepSeek-R1-Zero（纯 RL 版本）

这是 DeepSeek-R1 的前身，完全用 RL 训练，不用任何人工标注的推理步骤。关键技术是 GRPO（Group Relative Policy Optimization）。

GRPO 是什么？

GRPO 是 PPO（Proximal Policy Optimization）的简化版。PPO 是 RL 领域的经典算法，但计算开销大。GRPO 通过分组采样和相对策略优化，把计算量大幅降低。

🎯 类比：考试，单独改 vs 集体改

想象老师批改试卷：

  • PPO：每道题单独改，要算每道题的梯度，计算量大
  • GRPO：把 16 道题分成一组，一起改，只算组的平均梯度——计算量小很多

奖励设计：怎么告诉模型"做得好不好"？

RL 需要一个"奖励信号"告诉模型做得好不好。DeepSeek-R1-Zero 用的是基于规则的奖励：

  • 准确性奖励：答案对不对？数学题有标准答案，编程题能编译运行
  • 格式奖励：推理过程有没有按指定格式写？比如用 和 标签
关键设计：奖励完全基于"答案对不对"，不限制"怎么想"。这给了模型自由探索的空间。

训练过程：模型怎么"进化"？

DeepSeek-R1-Zero 的训练过程很有意思：

  • 初始阶段：模型像一张白纸，什么都不知道
  • 探索阶段：模型尝试各种方法，答错了就调整，答对了就强化
  • "顿悟时刻"：论文里提到一个有趣的瞬间——模型突然开始频繁用"wait"这个词，说明它学会了"停下来反思"
  • 收敛阶段：模型找到了一套稳定的推理策略，准确率从 15.6% 飙升到 77.9%

第二步：DeepSeek-R1（完整版本）

DeepSeek-R1-Zero 虽然推理能力强，但有问题：

  • 语言混合：中英文混着来，可读性差
  • 专注推理：只擅长数学、编程等推理任务，写作、问答等一般任务不行

为了解决这些问题，DeepSeek-R1 引入了多阶段训练框架：

阶段 1：冷启动 + RL

先用少量"冷启动数据"（人类对话式的推理过程）让模型学会"怎么说话"，再用 RL 提升推理能力。

阶段 2：拒绝采样 + SFT

用"拒绝采样"过滤掉不合适的回答，再用 SFT 训练，让模型学会写作、问答等一般任务。

阶段 3：偏好 RL

最后用模型奖励（不是规则奖励）对齐人类偏好，让模型更"有用"和"无害"。

🎯 类比：培养孩子，分阶段

  • 阶段 1：先让孩子学会"怎么说话"（冷启动）
  • 阶段 2：教孩子"怎么写作文"（SFT）
  • 阶段 3：让孩子学会"怎么帮人"（偏好 RL）

奖励模型：怎么让模型更"有用"？

DeepSeek-R1 用模型奖励（不是规则奖励）来对齐人类偏好。训练方法是：

  • 让 DeepSeek-V3（一个强大的模型）生成两个回答
  • 问 DeepSeek-V3 "哪个回答更好？"
  • 用 DeepSeek-V3 的判断作为奖励信号

这样做的好处是：

  • 不需要人工标注：让模型自己判断，省了人力
  • 能捕捉复杂偏好：人类偏好很难用规则描述，但模型能学会

Phase 4这技术能用吗？有什么限制？

一句话总结

DeepSeek-R1 是一个"自主进化"的推理模型：它通过强化学习自主学会推理能力，再通过多阶段训练对齐人类偏好，既有强大的推理能力，又适合实际应用。

什么时候该考虑用？

  • 你需要一个推理能力强的模型（数学、编程、逻辑推理）
  • 你有大量计算资源（RL 训练成本高）
  • 你想探索"超越人类"的推理方法

什么时候不该用？

  • 你的任务不需要复杂推理（比如简单的问答、摘要）——用 SFT 更快更便宜
  • 你的计算资源有限——RL 训练需要大量 GPU
  • 你需要完全可控的推理过程——RL 训练的过程是"黑盒"，难以解释

当前局限

  • 训练成本高——需要大量 GPU 和时间
  • 过程不可解释——模型是怎么学会推理的，很难追踪
  • 需要强大的基础模型——DeepSeek-V3 本身就很强

未来方向

  • 更高效的 RL 算法（进一步降低计算成本）
  • 探索其他领域的应用（比如视觉推理、多模态推理）
  • 提高训练过程的可解释性
